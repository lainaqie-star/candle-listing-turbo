# Candle Listing Turbo MVP

This repo contains a zero-dependency first version of a candle-focused productized service sales page.

## What is included

- `index.html`: the landing page
- `styles.css`: custom responsive styling
- `script.js`: CTA link config plus a lightweight listing preview demo

## How to use it

1. Open `index.html` directly in a browser for a quick preview.
2. Replace the placeholder checkout links in `script.js`.
3. Replace `hello@example.com` with your real inbox or payment URL.
4. Update the copy so the promise matches how you actually deliver.

## Fast customization checklist

- Brand name: edit `Candle Listing Turbo` in `index.html`
- Pricing: update the three package cards in `index.html`
- CTA destinations: update `siteConfig` in `script.js`
- Niche defaults: change the candle-specific hero, sample, and demo defaults if needed

## Suggested first sales flow

1. Prospect lands on the page
2. Prospect submits the simple order form or emails `lainaqie@gmail.com`
3. You reply with a Stripe or Gumroad payment link
4. After payment, send the intake template and deliver manually

## What to sell first

- Single candle listing rewrites
- Three-listing scent collection refreshes
- Seasonal candle launch bundles

## Fulfillment notes

Keep delivery in a repeatable template:

- Title options
- Keyword tag set
- Description opening
- Burn, gift, and scent FAQ block
- Image text suggestions
- One upsell recommendation for more listings or gift sets

## Launch advice

The point of this version is speed. Do not wait for a full SaaS build before talking to buyers.

If you can close buyers manually first, you will learn:

- which niches respond
- which deliverables people value
- what price point converts
- what can be automated later
